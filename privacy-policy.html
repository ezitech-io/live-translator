Live Translator — Privacy Policy

Developer: Doan Pham  ·  Contact: phampdoan@gmail.com  ·  Last updated: April 2026

Summary: Live Translator is designed to minimize data collection and does not run ads or usage analytics. Depending on the features you enable, data is processed locally in your browser and by third-party services you choose or use (including Soniox, OpenAI, Google Translate, and EziTranslate licensing/update APIs).

1. What Is Live Translator?

Live Translator is a browser extension that provides real-time speech transcription and translation in supported web apps (for example Google Meet, Zoom, Microsoft Teams, YouTube, WhatsApp Web, and other listed domains). Depending on enabled features, it can also provide subtitle mode, text-to-speech narration, inline AI assistance, and chat message insertion into supported meeting platforms.

2. Data We Process

Live Translator processes the minimum data required for the features you choose to use:

  • Audio data for transcription — During an active translation session, the extension captures tab audio (and optional microphone audio) and streams it to Soniox in real time for speech recognition and translation. This extension does not upload audio to any developer-operated server.
  • Transcript and prompt text — Transcribed/translated text can be processed by third-party services when you enable related features. For example, inline assistant requests may send transcript context and your prompt to OpenAI; narration features may send text to OpenAI TTS or Google Translate TTS.
  • Meeting chat actions — If you enable chat-send features, the extension can insert/send translated text into supported meeting chat inputs on the current page. The message is then handled by that platform under your account/session.
  • Local extension settings — The extension stores settings locally using chrome.storage.local, including configured API keys, language preferences, voice options, subtitle/display preferences, speaker labels, and saved context presets.
  • License and subscription metadata — If you activate premium features, license metadata (such as access token, activation identifier, expiry, and licensed customer name/email returned by the licensing API) is stored locally to maintain your license state.
  • Page context data (limited) — On supported pages (for example YouTube), title/description text may be read from the page DOM to help prefill context fields when that workflow is used.

We do not use your data for advertising or analytics: Live Translator does not run ad tracking, behavioral profiling, or sell your data. It also does not request or collect your full browsing history.

3. Transcript Handling and Export

Live transcript text displayed in the panel is kept in memory for the active session and is cleared when the session or panel is closed.

If you choose to export transcript files (.txt, .json, or .srt), files are generated and downloaded locally in your browser. You control where those files are stored after download.

4. Third-Party Services

Live Translator integrates with third-party services selected by feature usage:

  • Soniox — Real-time speech transcription/translation.
  • OpenAI — Optional inline assistant and optional TTS narration features.
  • Google Translate — Optional TTS narration provider.
  • EziTranslate API — License activation/verification and extension update checks.

Data sent to these services is governed by each provider's own terms and privacy policy. You are responsible for reviewing and accepting applicable third-party policies.

Live Translator is an independent product and is not affiliated with Google, Zoom, Microsoft, YouTube, WhatsApp, Soniox, or OpenAI.

5. Permissions Explanation

The extension requests the following permissions and access scopes:

  • storage — Store settings, API keys, context presets, and local license state.
  • activeTab — Open/toggle the panel in the active tab when you invoke the extension.
  • scripting — Inject and control panel scripts/styles on supported pages.
  • offscreen — Play narration audio in an offscreen document to reduce feedback loops during live transcription.
  • declarativeNetRequest — Apply scoped header rules required for specific supported platforms (for example WhatsApp Web compatibility).
  • Host permissions — Limited host access for https://translate.google.com/ (Google TTS) and https://web.whatsapp.com/* (platform compatibility).
  • Microphone / display-capture — Runtime browser prompts; used only when you explicitly start a translation session or related audio workflow.

The extension does not request universal host permission (<all_urls>). Content scripts run only on an explicit allowlist of supported domains defined in the manifest.

6. Security

Network communication to supported service providers uses encrypted HTTPS/WSS connections. Sensitive settings (such as API keys and license token data) are stored locally in extension storage on your device.

7. Data Retention

Session transcript state is temporary in memory. Locally saved settings, API keys, saved contexts, and license metadata remain on your device until you change/remove them, clear extension storage, or uninstall the extension.

Data processed by third-party providers is retained according to each provider's policy and is outside the direct control of this extension.

8. Children's Privacy

Live Translator is not directed at children under the age of 13 and does not knowingly collect any information from children.
